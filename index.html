About & Technical Information

Overview

HydroSim Global v2 is a browser-based, physically-based watershed hydrology simulator that runs a SWAT+-style daily water-balance model for any watershed on Earth. It automatically delineates the basin, pulls real climate, terrain, soil and land-cover data from open scientific APIs, derives model parameters from that data, simulates streamflow, and lets you calibrate and quantify uncertainty — entirely client-side, with no installation or server.

All computation happens locally in your browser. Data is fetched on demand from public providers listed below.

Model Physics (SWAT+ core)

Surface runoffSCS Curve Number (NEH-4) with dynamic antecedent moisture (CN1/CN2/CN3)
EvapotranspirationFAO-56 Penman-Monteith (ERA5), Priestley-Taylor, or Hargreaves-Samani; FAO-56 Ks water-stress
SnowSeasonal degree-day melt (SMFMX/SMFMN) with sublimation
Soil waterField-capacity / saturation / wilting-point storage with gravity drainage
GroundwaterShallow aquifer, delayed recharge, α-recession baseflow, deep-loss sink
Channel routingMuskingum (K, X); Variable Storage option
SedimentMUSLE (Williams 1975)
RadiationFAO-56 extraterrestrial radiation by latitude & day-of-year

Every run reports a water-balance closure check (P = ET + Q + deep loss + ΔStorage), typically within <2%.

Live Data Sources

Watershed boundaryMERIT-Hydro / HydroSHEDS via Global Watersheds API (mghydro.com); HydroBASINS L8 & USGS NLDI fallbacks
ClimateERA5 reanalysis via Open-Meteo (P, T, PET, snowfall) — spatially averaged over basin grid
Elevation (DEM)SRTM 90 m via OpenTopoData — 25-point basin sampling
SoilISRIC SoilGrids v2 (clay, sand, AWC, Ksat) — multi-point basin average
Land coverESA WorldCover classification (climate-terrain proxy)
GeocodingNominatim (OpenStreetMap)
Base mapsCARTO, Esri World Imagery, OpenTopoMap

Calibration & Uncertainty

SamplingLatin Hypercube (stratified) — SUFI-2, PSO, GLUE, ParaSol
ObjectivesNSE, KGE′ (2012), log-NSE, NSE+logNSE, PBIAS, R²
Validation70/30 temporal split; 1-year warmup (spin-up) excluded
UncertaintyGLUE 95PPU band with P-factor & R-factor
Sensitivityt-stat multiple-regression ranking of parameters
SignaturesBaseflow index, Richards-Baker flashiness, flow-duration Q5/Q95

Reference Discharge & Limitations

When no gauge record is loaded, the reference streamflow used for calibration is a Budyko water-balance benchmark (Fu/Zhang equation) driving the SWAT engine with a perturbed parameter set — a physically-constrained stand-in, not observed data. For rigorous studies, substitute real gauge records (e.g. GRDC or USGS NWIS). Delineation snaps to the clicked outlet, so results depend on placing the point on the intended river reach; arid/ephemeral basins are inherently harder to calibrate and auto-switch to KGE. Data APIs are called from your browser and depend on the providers' availability and CORS policy; if any are unreachable, the model falls back to physically-reasonable estimates and flags this in the status badges.

Help — How to Use

Quick Start (4 steps)

1
Pick a watershed. Click anywhere on the map, search a river/city/basin, type coordinates, or hit a benchmark basin (Amazon, Rhine, …).
2
Fetch Real Data. Press ⬇ Fetch Real Data. The app delineates the basin and pulls ERA5, SRTM, SoilGrids & land cover, then auto-fills all SWAT+ parameters. Watch the status badges turn green.
3
Run the simulation. Choose period & ET method, then press ▶ Run SWAT+ Simulation. Streamflow, water balance and metrics animate in.
4
Calibrate & export. Open Calibration to auto-tune parameters, view Results for diagnostics, and Export to preview SWAT output files.

The Tabs

SimulationMain screen — pick location, set parameters, run the model, see live charts
CalibrationAuto-tune parameters (SUFI-2/PSO/GLUE), set ranges, watch dotty plots & sensitivity, get calibration + validation NSE
ResultsMonthly sim-vs-obs, flow-duration curve, annual water balance, obs-vs-sim scatter
ExportPreview SWAT+ output.rch / .hru / .sub, climate & soil CSVs, best-parameter file, 95PPU band

Reading the Metrics

NSENash-Sutcliffe. >0.75 very good · 0.5–0.75 satisfactory · <0 worse than the mean
KGEKling-Gupta. 1 = perfect; balances correlation, bias & variability
logNSENSE on log-flows — rewards low-flow / baseflow accuracy
PBIAS% volume bias. |PBIAS|<10 very good; + means model under-predicts
P-factorFraction of obs inside the 95% uncertainty band (aim >0.7)
R-factorBand width ÷ obs std-dev (aim <1.5 — tight but realistic)

Tips & Troubleshooting

›
Place the outlet on the river you want — delineation snaps to that point, so a tributary click gives a small sub-basin.
›
A red status badge means an API was unreachable; the model still runs on physically-reasonable estimates.
›
Large basins (>50,000 km²) delineate at lower precision and may take 10–20 s on first request.
›
Adjust any parameter slider manually, then re-run — or let Calibration find the best set for you.
›
Switch base maps (🌑 dark / 🛰 satellite / 🏔 terrain) and toggle the basin outline (🌀) from the map controls.
